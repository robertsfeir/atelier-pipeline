---

## Summary

Overall code health is solid: the brain server is cleanly modularized, tests exist per module, and the hook layer takes graceful-degradation seriously. The dominant defect class is **systemic duplication of state-file parsing**: the same fragile `PIPELINE_STATUS: {[^}]*}` regex is reimplemented across 5 shell scripts with no shared helper, and `hydrate-telemetry.mjs` has two nearly-identical inlined loops in `main()` that shadow the extracted `hydrateSubagentFile` / `hydrateEvaFile` helpers. A handful of security and defensive-programming gaps (unauthenticated UI serving the API token, LLM response parsing without null checks, inconsistent HTML escaping) round out the findings, alongside one genuine parameter-ordering confusable in a production SQL insert.

## Positive Observations

1. **`brain/lib/embed.mjs:27-90`** — `getEmbedding` is a defensive-programming exemplar: explicit retry table (`BACKOFF_MS = [1000, 2000, 4000]`), explicit `isRetryable` predicate, branch for network-error detection by `err.code`, and shape validation of the response before touching `data.data[0].embedding`. Exactly the null-guarding missing in conflict.mjs and consolidation.mjs.
2. **`brain/lib/rest-api.mjs:439-457`** — `readBody` caps request size at 1MB (`MAX_BODY_SIZE`), destroys the request when exceeded, and resolves cleanly otherwise. Simple, correct, and something many Node REST servers get wrong.
3. **`brain/lib/tools.mjs:485-498`** — `checkSupersedeCycle` uses a recursive CTE with an explicit `chain.depth < 20` guard. Cycle detection for a graph relation with a bounded-depth escape is the right instinct.
4. **`brain/lib/static.mjs:46-50`** — Directory-traversal guard on `filePath.startsWith(UI_DIR)` is present. The intent is explicit and the code bothers to check.
5. **`brain/lib/tools.mjs:537-551`** — Batch `superseded_by` lookup via a single `SELECT ... WHERE target_id = ANY($1)` followed by Map construction, replacing what would otherwise be an N+1 query inside the chain loop. The inline comment even calls out the optimization.
6. **`.claude/hooks/enforce-sequencing.sh:45-50`** — Snapshots `pipeline-state.md` to a tempfile under a `trap 'rm -f ...' EXIT` before parsing. The author clearly thought about the "Eva mid-write" race and defended against it.
7. **`brain/lib/db.mjs:43-186`** — `runMigrations` wraps each migration in its own `try/catch` so one migration's failure cannot block the others, and each migration is guarded by an idempotency check before running the SQL.
8. **`brain/lib/crash-guards.mjs:60-61`** — `process.removeAllListeners('uncaughtException')` before installing the brain's own handler, with a specific inline comment explaining why (test runners register their own handlers that would otherwise fire first).
9. **`tests/brain/rest-api.test.mjs:15-56`** — Hand-rolled `createMockReq` / `createMockRes` that emits body in a controllable way and exposes `get json()` convenience. Correct minimum surface area.
10. **`brain/lib/consolidation.mjs:15-57`** — `createUnionFind()` is a clean, self-contained implementation with path compression (`parent.set(x, find(parent.get(x)))`) and union-by-rank. Correct, compact, and well-isolated from the SQL and LLM code around it.
