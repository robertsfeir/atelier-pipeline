## Executive Summary

The atelier-pipeline is architecturally coherent and its enforcement pyramid is real — the three-layer PreToolUse gate, the hook registration model, and the brain capture architecture all reflect deliberate, documented design decisions that hold up under independent scrutiny. The codebase has no TODO/FIXME markers in production code, all SQL is parameterized, the hook layer takes graceful-degradation seriously, and the ADR-0033 test suite is a model of Roz-first TDD discipline.

The four most consequential gaps form two clusters. **Cluster 1 (brain capture drift):** the `SOURCE_AGENTS` and `SOURCE_PHASES` Zod enums in `brain/lib/config.mjs` have been out of sync with the brain-extractor persona mapping table since the scope was extended — every `agent_capture` call from robert/robert-spec/sable/sable-ux/ellis/sentinel/darwin/deps currently fails Zod validation and is silently discarded. Four independent agents (Cal, Colby, Roz, Robert) converged on this finding from different angles. **Cluster 2 (MCP auth gap):** the brain HTTP server's MCP endpoint is unauthenticated while simultaneously the static handler injects the master Bearer token into every served HTML page. Two agents (Sentinel, Poirot) reached this from security and code-ordering directions respectively. Together, any localhost-reachable process can read/write all institutional memory and trigger paid API calls.

The recommended first two actions — fix the enum + add migration, then gate the MCP endpoint behind auth — close the highest-impact gaps with surgical, well-bounded changes.
